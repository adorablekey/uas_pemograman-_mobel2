# Penjelasan Prototype Aplikasi Badminton  Booking

Prototype ini merupakan rancangan antarmuka (UI) aplikasi Badminton Booking yang dibuat menggunakan Figma. Aplikasi ini bertujuan untuk memudahkan pengguna dalam melakukan pemesanan lapangan badminton secara online tanpa harus datang langsung ke lokasi.

Prototype ini terdiri dari beberapa halaman yang saling terhubung sehingga menggambarkan alur penggunaan aplikasi mulai dari login hingga konfirmasi pemesanan.

# 1. Halaman Login
Halaman pertama digunakan sebagai proses autentikasi pengguna sebelum mengakses aplikasi.

Komponen yang terdapat pada halaman ini
* Judul aplikasi "Badminton Court Booking"
* Input Username
* Input Password
* Tombol Masuk
Pengguna diminta memasukkan username dan password yang telah terdaftar. Setelah data diisi dengan benar, pengguna dapat menekan tombol Masuk untuk menuju halaman utama aplikasi.

# 2. Halaman Dashboard 
Halaman utama yang menampilkan menu serta daftar jadwal lapangan.

Komponen:
* Tombol **Riwayat**
* Tombol **Keluar**
* Jadwal lapangan
* Pilihan lapangan

Pada halaman ini pengguna dapat melihat jadwal yang tersedia, memilih lapangan yang ingin disewa, melihat riwayat pemesanan sebelumnya, ataupun keluar dari aplikasi.

# 3. Halaman Daftar Lapangan
Menampilkan seluruh pilihan lapangan beserta harga sewanya.

Informasi yang ditampilkan
* Nama lapangan
* Harga per jam
* Status ketersediaan
* Tombol **Booking**

Pengguna dapat membandingkan beberapa lapangan yang tersedia. Jika lapangan masih tersedia, pengguna dapat menekan tombol **Booking** untuk melanjutkan proses pemesanan.

Contoh:
* Lapangan A (Vinyl) : Rp50.000/jam
* Lapangan B (Kayu) : Rp45.000/jam
* Lapangan C : Rp30.000/jam

# 4. Halaman Detail Pemesanan
Menampilkan rincian lapangan yang dipilih sebelum melakukan pembayaran.

Informasi yang tersedia
* Nama lapangan
* Jenis lapangan
* Harga sewa
* Jam operasional
* Durasi sewa
* Jumlah jam
* Tombol Hitung Total Harga
* Total pembayaran
* Tombol Pesan Sekarang
Pada tahap ini pengguna menentukan lama waktu penyewaan. Sistem akan menghitung total biaya berdasarkan harga sewa per jam dikalikan dengan durasi yang dipilih.

Contoh:

* Harga: Rp50.000/jam
* Durasi: 2 jam
* Total pembayaran: Rp100.000

Apabila sudah sesuai, pengguna dapat menekan tombol **Pesan Sekarang**.

# 5. Halaman Konfirmasi Pesanan
Sebagai tahap terakhir sebelum transaksi selesai.

Informasi yang ditampilkan:
* Nama lapangan
* Tipe lapangan
* Waktu sewa
* Kapasitas
* Fasilitas
* Durasi
* Total pembayaran

Tombol
* Batal
* Selesai
Halaman ini berfungsi sebagai konfirmasi akhir agar pengguna dapat memastikan kembali seluruh data pemesanan sebelum transaksi diselesaikan.

# Alur Penggunaan Aplikasi

1. Pengguna membuka aplikasi.
2. Login menggunakan username dan password.
3. Masuk ke halaman utama.
4. Memilih lapangan yang tersedia.
5. Menekan tombol **Booking**.
6. Menentukan durasi sewa.
7. Sistem menghitung total biaya.
8. Menekan tombol **Pesan Sekarang**.
9. Melakukan konfirmasi pemesanan.
10. Pemesanan berhasil diselesaikan.

# Tujuan Pembuatan Prototype

Prototype ini dibuat untuk memberikan gambaran mengenai tampilan dan alur kerja aplikasi sebelum masuk ke tahap pengembangan sistem yang sebenarnya. Dengan adanya prototype, pengembang dapat mengevaluasi desain antarmuka, kemudahan penggunaan (usability), serta memastikan setiap proses pemesanan berjalan sesuai kebutuhan pengguna.

# Kesimpulan

Prototype Badminton Booking berhasil menggambarkan proses pemesanan lapangan badminton secara sederhana dan mudah dipahami. Mulai dari proses login, pemilihan lapangan, perhitungan biaya sewa, hingga konfirmasi pemesanan telah dirancang dengan alur yang sistematis sehingga diharapkan dapat memberikan pengalaman pengguna yang lebih efektif, efisien, dan nyaman dalam melakukan reservasi lapangan badminton.
